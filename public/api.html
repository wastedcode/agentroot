API Reference

AgentRoot's REST API is open and requires no authentication. Base URL: https://agentroot.io. All responses are JSON. Rate limits may apply in the future — please be reasonable. Questions? Check the TXT spec or open an issue.

GET /api/agents
List all active agents in the registry. Supports text search and filtering by capabilities, category, and status. Returns agents verified via DNS — no pending or failed entries.

Query Parameters

ParameterTypeDescription
qstringFull-text search across name, domain, description, capabilities
capsstringFilter by capabilities (AND logic). Comma-separated: caps=research,finance
categorystringFilter by category: research, finance, devtools, data, communication, automation, media, security
statusstringFilter by status (active, pending, failed). Default: active only

Example Request

# Get all agents curl https://agentroot.io/api/agents # Search by keyword curl "https://agentroot.io/api/agents?q=finance" # Filter by multiple caps (AND logic) curl "https://agentroot.io/api/agents?caps=research,finance" # Filter by category curl "https://agentroot.io/api/agents?category=devtools"

Response

{ "agents": [ { "id": 1, "domain": "example.com", "name": "MyAgent", "description": "A helpful AI agent", "capabilities": "research,summarize", "endpoint": "https://example.com/a2a", "protocol": "a2a", "card_url": "https://example.com/.well-known/agent-card.json", "category": "research", "status": "active", "first_verified": "2026-03-01T12:00:00Z", "last_verified": "2026-03-01T18:00:00Z", "last_checked": "2026-03-01T18:00:00Z", "submitted_at": "2026-03-01T12:00:00Z", "raw_record": "v=agent1 name=MyAgent caps=research,summarize endpoint=https://example.com/a2a" } ], "count": 1 }
GET /api/agents/:domain
Get a single agent by domain name. Returns all fields including raw DNS record. Works for any status (active, pending, failed).

Example Request

curl https://agentroot.io/api/agents/example.com

Response

{ "agent": { // ... same fields as above } } # 404 if not found: { "error": "Agent not found" }
POST /api/submit
Submit a domain for indexing. AgentRoot performs a live DNS lookup for _agent.<domain>. If found, the agent is indexed immediately and returns status: "active". If not found, the domain is stored as pending and instructions are returned.

Request Body

FieldTypeDescription
domainstringDomain to submit (e.g., example.com). Required.

Example Request

curl -X POST https://agentroot.io/api/submit \ -H "Content-Type: application/json" \ -d '{"domain": "example.com"}'

Response — DNS Record Found

{ "success": true, "status": "active", "message": "_agent.example.com TXT record found and indexed!", "agent": { /* full agent object */ } }

Response — No DNS Record Found

{ "success": false, "status": "pending", "message": "No _agent.example.com TXT record found. Add one to get indexed!", "instructions": { "record": "_agent.example.com", "type": "TXT", "value": "v=agent1 name=YourAgent caps=your,capabilities endpoint=https://example.com/a2a", "spec": "https://agentroot.io/spec.html" } } # Error cases: { "error": "domain is required" } # 400 { "error": "Invalid domain format" } # 400
GET /api/stats
Registry statistics — total agents, active/pending/failed counts, last verification run, and breakdown by TLD.

Example Request

curl https://agentroot.io/api/stats

Response

{ "total": 42, "active": 38, "pending": 3, "failed": 1, "lastVerification": "2026-03-01T18:00:00.000Z", "byTld": { "com": 20, "io": 12, "ai": 6 } }
GET /api/categories
List all capability categories with agent counts. Categories are derived from agents' caps= field at registration time.

Example Request

curl https://agentroot.io/api/categories

Response

{ "categories": [ { "id": "research", "label": "Research", "emoji": "🔍", "desc": "Search, browse, and knowledge agents", "count": 8 }, { "id": "finance", "label": "Finance", "emoji": "💰", "desc": "Trading, crypto, and payments agents", "count": 5 }, // ... 6 more categories ] }
GET /api/agents/:domain/badge
Returns an SVG badge showing whether a domain is listed on AgentRoot. Useful for embedding in READMEs, websites, or agent cards.

Example Request

curl https://agentroot.io/api/agents/example.com/badge

Embed Code

<!-- In HTML --> <a href="https://agentroot.io/agents/example.com"> <img src="https://agentroot.io/api/agents/example.com/badge" alt="Listed on AgentRoot"> </a> # In Markdown (README) [![Listed on AgentRoot](https://agentroot.io/api/agents/example.com/badge)](https://agentroot.io/agents/example.com)

GET /.well-known/agent-card.json
AgentRoot's own agent card — its declaration of capabilities, endpoint, and skills in the A2A agent card format. Any agent that implements A2A can discover AgentRoot's capabilities through this endpoint.

Example Request

curl https://agentroot.io/.well-known/agent-card.json

Response

{ "name": "AgentRoot", "description": "DNS-native agent registry for the agentic internet...", "version": "1.0.0", "endpoint": "https://agentroot.io/a2a", "protocol": "a2a", "capabilities": ["agent_discovery", "registry_query", "dns_verification", "agent_submission"], "skills": [ { "name": "discover_agents", "description": "Find agents by capability, name, or domain" }, { "name": "submit_domain", "description": "Submit a domain for _agent DNS record indexing" }, { "name": "verify_domain", "description": "Re-verify a domain's _agent DNS record" }, { "name": "registry_stats", "description": "Get statistics about the indexed agent registry" } ] }

Calling AgentRoot via A2A

# Discover agents with specific capabilities curl -X POST https://agentroot.io/a2a \ -H "Content-Type: application/json" \ -d '{ "id": "task-001", "skill": "discover_agents", "input": { "capabilities": "research,finance" } }'
