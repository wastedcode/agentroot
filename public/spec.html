DNS TXT Record Specifications v1

AgentRoot defines two DNS TXT record formats: _agent for autonomous AI agents, and _skill for callable tools. Any domain can publish one or both. AgentRoot reads them and indexes the agentic internet.

_agent
Autonomous agents
Entities with memory, state, and conversation. They talk to you. Register at _agent.yourdomain.com.
_skill
Callable skills / tools
Stateless request/response tools. You call them. MCP, OpenAPI, GraphQL, REST, gRPC. Register at _skill.yourdomain.com.

Part 1: _agent Record

Overview

Add a TXT record to your DNS at _agent.<yourdomain>. The record encodes your agent's name, capabilities, endpoint, and other metadata in a simple key=value format. No web server required for discovery — just DNS.

Subdomain Support

The _agent TXT record works on any domain or subdomain. A single organization can register multiple agents under one root domain by using subdomains — one record per agent.

How the lookup works: AgentRoot always prepends _agent. to whatever domain you submit. So if your agent lives on a subdomain, submit the full subdomain — e.g. radar.yourdomain.com, not yourdomain.com.

Multi-agent examples

DNS Record What to submit Use case
_agent.agentroot.io agentroot.io The registry itself
_agent.radar.yourdomain.com radar.yourdomain.com A signal intelligence agent
_agent.assistant.yourdomain.com assistant.yourdomain.com A personal assistant agent

Each subdomain gets its own _agent TXT record and appears as a separate entry in the registry. There's no limit on how many agents an org can register this way.

Record Format

The value is a space-separated list of key=value pairs starting with the version prefix v=agent1.

_agent.yourdomain.com TXT "v=agent1 name=AgentName caps=cap1,cap2 endpoint=https://yourdomain.com/a2a proto=a2a desc=Short\ description"

Fields

Field Required Description Example
v Required Version identifier. Must be agent1 for this spec. v=agent1
name Required Human-readable name of the agent. No spaces — use backslash-space for multi-word. name=MyAgent
caps Required Comma-separated list of capabilities. Use lowercase, underscores OK. No spaces. caps=search,summarize,translate
endpoint Optional URL of the agent's A2A or API endpoint. HTTPS recommended. endpoint=https://yourdomain.com/a2a
proto Optional Protocol spoken by the endpoint. Defaults to a2a. proto=a2a
desc Optional Short description. Use backslash-space for multi-word values. desc=Searches\ and\ summarizes\ the\ web
card Optional URL to an agent-card.json file (A2A standard) with full agent metadata. card=https://yourdomain.com/.well-known/agent-card.json
Note on multi-word values:** DNS TXT records don't support quoted sub-values. Use a backslash followed by a space (\) within a value to encode spaces. AgentRoot decodes these when indexing.

Examples

Minimal (required fields only)

_agent.mybot.com TXT "v=agent1 name=MyBot caps=chat,search"

Full record

_agent.mybot.com TXT "v=agent1 name=My\ Research\ Bot caps=research,summarize,cite endpoint=https://mybot.com/a2a proto=a2a desc=Academic\ research\ assistant card=https://mybot.com/.well-known/agent-card.json"

Cloudflare DNS dashboard

Type: TXT Name: _agent Value: v=agent1 name=MyAgent caps=summarize,translate endpoint=https://myagent.com/a2a

Route53 / CLI

aws route53 change-resource-record-sets --hosted-zone-id YOUR_ZONE_ID \ --change-batch '{ "Changes": [{ "Action": "CREATE", "ResourceRecordSet": { "Name": "_agent.yourdomain.com", "Type": "TXT", "TTL": 300, "ResourceRecords": [{"Value": "\"v=agent1 name=MyAgent caps=search,summarize\""}] } }] }'

Verification

After adding the record, verify it works:

dig TXT _agent.yourdomain.com +short # or nslookup -type=TXT _agent.yourdomain.com

Then submit your domain to AgentRoot. We'll do a live DNS lookup and index you immediately.

How AgentRoot Uses It

AgentRoot resolves _agent.<domain> TXT records via DNS on submission and every 6 hours thereafter. If a record disappears, the agent is marked as offline but not deleted. AgentRoot never hits your web server for discovery — DNS only.

Questions / Feedback

This spec is maintained by AgentRoot. To propose changes, open an issue or contact Clawd.

Part 2: _skill Record

The _skill TXT record publishes a callable tool — an MCP server, OpenAPI endpoint, GraphQL API, REST endpoint, or gRPC service. Any agent or developer can query AgentRoot to discover skills and auto-configure tool calls.

Overview

Skills are stateless callable tools, not entities. They don't have conversations — you call them with inputs and get outputs. The proto field tells agents how to call the skill, and schema points to a machine-readable definition (OpenAPI spec, MCP manifest) for automatic integration.

Record Format

_skill.yourdomain.com TXT "v=skill1 name=MySkill proto=mcp caps=search,query endpoint=https://yourdomain.com/mcp schema=https://yourdomain.com/mcp/manifest.json auth=apikey cost=free desc=Short\ description"

Fields

Field Required Description Example
v Required Version. Must be skill1. v=skill1
name Required Human-readable name of the skill. name=Stripe
proto Required Protocol. See values below. This is the key field — tells agents HOW to call the skill. proto=mcp
caps Required Comma-separated list of callable methods or operations. caps=search,trending,comments
endpoint Optional Base URL where the skill is reachable. endpoint=https://mcp.reddit.com
schema Optional URL to machine-readable schema. For MCP: manifest JSON. For OpenAPI: .json/.yaml spec. Enables automatic tool integration. schema=https://mcp.reddit.com/manifest.json
auth Optional Auth model. Helps agents know what credentials they need. Values: none, apikey, oauth2, bearer. auth=apikey
cost Optional Pricing signal. Values: free, freemium, paid. cost=free
desc Optional Short description (≤120 chars). Use backslash-space for multi-word. desc=Searches\ Reddit\ posts

Protocol Values

ValueProtocolSchema field should point to
mcpModel Context ProtocolMCP server manifest JSON
openapiOpenAPI / Swagger.json or .yaml spec file
graphqlGraphQLIntrospection endpoint or SDL URL
restGeneric RESTAny docs URL
grpcgRPC.proto file URL
Recommendation: Always include a schema URL for mcp and openapi skills. This enables agents to auto-configure tool calls without human integration. Skills without a schema URL are shown in the registry but marked as requiring manual integration.

Examples

MCP Server

_skill.reddit.com TXT "v=skill1 name=Reddit proto=mcp caps=search,trending,comments endpoint=https://mcp.reddit.com schema=https://mcp.reddit.com/manifest.json auth=oauth2 cost=free"

OpenAPI (REST)

_skill.stripe.com TXT "v=skill1 name=Stripe proto=openapi caps=charge,refund,customer,subscription endpoint=https://api.stripe.com/v1 schema=https://raw.githubusercontent.com/stripe/openapi/master/openapi/spec3.json auth=apikey cost=paid"

GraphQL

_skill.github.com TXT "v=skill1 name=GitHub proto=graphql caps=repos,issues,prs,search endpoint=https://api.github.com/graphql schema=https://docs.github.com/public/schema.docs.graphql auth=bearer cost=freemium"

A Domain Can Have Both

A domain can publish both an _agent and a _skill record simultaneously. AgentRoot indexes both. Example: a service might expose an autonomous agent and a callable API.

; Same domain, two records — totally valid _agent.yourdomain.com TXT "v=agent1 name=MyAgent caps=research,summarize endpoint=https://yourdomain.com/a2a" _skill.yourdomain.com TXT "v=skill1 name=MyAPI proto=openapi caps=search,summarize endpoint=https://yourdomain.com/api schema=https://yourdomain.com/openapi.json auth=apikey cost=freemium"
