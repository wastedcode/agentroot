The _agent and _skill DNS TXT record format. Version 1. Open standard — implement it yourself, fork it, improve it.

// overview

Two record types

_agent

Autonomous AI agent

An agent has memory, makes decisions, and acts on your behalf. It's an entity, not just a function call. Use _agent for things that think.

_skill

Callable tool or API

A skill is a stateless capability — a function you can call. Use _skill for APIs, MCP servers, and tools that do one thing well.

A domain can have both. _agent.example.com and _skill.example.com are independent records.

// _agent format

Agent record

# DNS record _agent.yourdomain.com IN TXT "v=agent1 name=MyAgent caps=search,analyze endpoint=https://yourdomain.com/api proto=rest desc=Short\ description"
FieldRequiredDescriptionExample
vrequiredVersion. Always agent1 for this version.v=agent1
namerequiredShort display name. No spaces.name=Radar
capsrequiredComma-separated capability list. Use lowercase, no spaces.caps=search,analyze,alert
endpointoptionalURL where this agent can be called.endpoint=https://example.com/api
protooptionalProtocol the agent speaks. Any value accepted.proto=rest
descoptionalShort description. Escape spaces with \ desc=Signal\ intelligence
cardoptionalURL to a full agent-card.json for rich metadata.card=https://example.com/.well-known/agent-card.json
// _skill format

Skill record

# DNS record _skill.yourdomain.com IN TXT "v=skill1 name=MyAPI proto=openapi caps=search,query endpoint=https://yourdomain.com/api schema=https://yourdomain.com/openapi.json auth=apikey cost=free"
FieldRequiredDescriptionExample
vrequiredVersion. Always skill1.v=skill1
namerequiredShort display name.name=StripeAPI
protorequiredProtocol. Accepted values: mcp, openapi, rest, graphql, a2a, grpc, or any custom value.proto=mcp
capsrequiredComma-separated capabilities.caps=charge,refund
endpointoptionalURL to call this skill.endpoint=https://api.example.com
schemaoptionalURL to machine-readable spec (OpenAPI JSON, MCP manifest).schema=https://example.com/openapi.json
authoptionalAuth model. Values: none, apikey, oauth2, bearer.auth=apikey
costoptionalPricing model. Values: free, paid, freemium.cost=free
descoptionalShort description. Escape spaces with \ desc=Payment\ processing\ API
// subdomains

One domain, many agents

Use subdomains to register multiple agents or skills under one organization. Submit each subdomain separately to AgentRoot.

# Multiple agents under one org _agent.example.com → main agent _agent.radar.example.com → signal intelligence agent _agent.assistant.example.com → personal assistant # Skills alongside agents _skill.api.example.com → public API as a skill _skill.search.example.com → search capability
// examples

Real-world examples

Signal intelligence agent
"v=agent1 name=Radar caps=trend_scan,signal_score,breakthrough_alert endpoint=https://radar.example.com/api proto=rest desc=Early\ trend\ detection\ for\ AI\ crypto\ finance"
MCP skill (tool server)
"v=skill1 name=GitHubSearch proto=mcp caps=repo_search,trending,code_search endpoint=https://mcp.example.com schema=https://mcp.example.com/manifest.json auth=apikey cost=free"
OpenAPI skill
"v=skill1 name=WeatherAPI proto=openapi caps=current,forecast,alerts endpoint=https://api.example.com/v1 schema=https://api.example.com/openapi.json auth=none cost=freemium"
