$ man agentroot

Register your agent or skill in 3 steps. No accounts. No API keys. No approval process. Just DNS.

// registration
01

Add a TXT record to your DNS

Go to your DNS provider and add a TXT record. The record name is _agent.yourdomain.com (for agents) or _skill.yourdomain.com (for tools/skills).

# For an agent _agent.yourdomain.com TXT "v=agent1 name=MyAgent caps=search,analyze endpoint=https://yourdomain.com/api proto=rest" # For a skill (callable tool) _skill.yourdomain.com TXT "v=skill1 name=MyAPI proto=openapi caps=search,summarize endpoint=https://yourdomain.com/api"

Works with any DNS provider:

Cloudflare Route 53 Namecheap Google Domains GoDaddy Porkbun Any other
02

Submit your domain

Tell AgentRoot to look you up. We'll query _agent.yourdomain.com (and _skill.yourdomain.com) via DNS, verify the record exists, and add you to the registry. No account needed.

# Via the web UI → agentroot.io/submit.html # Or via the API curl -X POST https://agentroot.io/api/submit \ -H "Content-Type: application/json" \ -d '{"domain": "yourdomain.com"}'

DNS propagation can take up to 24h, but usually resolves in minutes.

03

Get discovered

Your agent or skill is now publicly listed in the AgentRoot registry. Anyone can find it — developers browsing the UI, or other agents querying the API to build their tool set.

# Anyone can query your agent curl https://agentroot.io/api/agents/yourdomain.com # Or discover all agents + skills at once curl https://agentroot.io/api/discover # Your agent-card is also available curl https://yourdomain.com/.well-known/agent-card.json

// subdomains

Multiple agents per domain

A single organization can register multiple agents and skills using subdomains. Submit each subdomain separately.

# Register each subdomain separately _agent.agentroot.io → AgentRoot (the registry itself) _agent.radar.yourdomain.com → Radar (signal intelligence agent) _skill.api.yourdomain.com → your public API as a skill
